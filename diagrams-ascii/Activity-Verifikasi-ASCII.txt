PENJELASAN ALUR:

1. LOGIN VERIFIKATOR
   └─ Verifikator login ke sistem dengan kredensial

2. DASHBOARD VERIFIKATOR
   └─ Melihat ringkasan data yang perlu diverifikasi

3. LIHAT DAFTAR PENGAJUAN
   ├─ Menampilkan sekolah dengan status_verifikasi = 'pending'
   ├─ Diurutkan berdasarkan tanggal pengajuan
   └─ Menampilkan informasi dasar sekolah

4. PILIH SEKOLAH
   └─ Verifikator memilih sekolah yang akan diverifikasi

5. LIHAT DETAIL DATA
   ├─ Identitas Sekolah
   │  ├─ NPSN, nama, alamat
   │  ├─ Kepala sekolah
   │  └─ Status akreditasi
   │
   ├─ Data Sosekbud
   │  ├─ Kondisi geografis
   │  ├─ Kondisi sosial ekonomi budaya
   │  └─ Akses transportasi
   │
   ├─ Data Bantuan
   │  ├─ Status bantuan
   │  ├─ Nama lembaga
   │  └─ Keterangan bantuan
   │
   ├─ Fasilitas TIK
   │  ├─ Status listrik
   │  ├─ Jumlah komputer
   │  ├─ Status internet
   │  └─ Laboratorium komputer
   │
   └─ Data Guru
      ├─ Identitas guru
      ├─ Kompetensi TIK
      ├─ Riwayat pelatihan
      └─ Kebutuhan pelatihan

6. VALIDASI DATA
   └─ Verifikator memeriksa kelengkapan dan kebenaran data

7. KEPUTUSAN VERIFIKASI

   A. JIKA DATA VALID (APPROVED)
      ├─ Status diubah menjadi 'approved'
      ├─ Keterangan verifikasi: "Data valid dan lengkap"
      └─ Notifikasi ke operator: "Data Anda telah disetujui"

   B. JIKA DATA TIDAK VALID (REJECTED)
      ├─ Status diubah menjadi 'rejected'
      ├─ Input catatan perbaikan yang detail
      └─ Notifikasi ke operator: "Data ditolak, silakan perbaiki"

8. NOTIFIKASI
   ├─ Email notification (optional)
   ├─ In-app notification
   └─ Update status di dashboard operator

9. ITERASI
   └─ Verifikator dapat melanjutkan ke data berikutnya

STATUS VERIFIKASI:

┌──────────┬────────────────────────────────────────────────────────────────┐
│  Status  │                        Deskripsi                               │
├──────────┼────────────────────────────────────────────────────────────────┤
│ pending  │ Menunggu verifikasi dari verifikator                          │
│ approved │ Data telah diverifikasi dan disetujui                         │
│ rejected │ Data ditolak, perlu perbaikan sesuai catatan                  │
└──────────┴────────────────────────────────────────────────────────────────┘

KRITERIA VALIDASI:

✓ IDENTITAS SEKOLAH
  ├─ NPSN valid dan sesuai format
  ├─ Nama sekolah lengkap
  ├─ Alamat jelas dan detail
  ├─ Data kepala sekolah lengkap
  └─ Status akreditasi sesuai

✓ SOSIAL EKONOMI BUDAYA
  ├─ Kondisi geografis terisi
  ├─ Kondisi sosekbud terisi
  └─ Akses transportasi terisi

✓ BANTUAN SEKOLAH
  ├─ Status bantuan jelas
  ├─ Nama lembaga valid (jika ada bantuan)
  └─ Keterangan bantuan detail

✓ FASILITAS TIK
  ├─ Status listrik terisi
  ├─ Data komputer lengkap
  ├─ Status internet terisi
  └─ Data lab komputer (jika ada)

✓ DATA GURU
  ├─ Minimal 1 guru terinput
  ├─ Data identitas lengkap
  ├─ Kompetensi TIK terisi
  └─ Data pelatihan/kebutuhan terisi

CONTOH CATATAN PERBAIKAN:

❌ REJECTED - Contoh Catatan:
   "Data kepala sekolah belum lengkap. Mohon dilengkapi:
    - Nomor HP kepala sekolah
    - Foto kepala sekolah
    
    Data fasilitas TIK:
    - Mohon lengkapi sumber listrik
    - Bandwidth internet belum terisi
    
    Data guru:
    - Guru dengan NIP 123456 belum ada data pelatihan"

✓ APPROVED - Contoh Keterangan:
   "Data sekolah telah lengkap dan valid. 
    Terverifikasi pada: 15 Februari 2026"
