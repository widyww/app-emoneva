PENJELASAN ALUR:

1. USER AKSES LOGIN PAGE
   └─ User membuka halaman login di browser

2. BROWSER REQUEST LOGIN PAGE
   └─ GET /login ke AuthController

3. AUTH CONTROLLER RETURN FORM
   └─ Menampilkan form login (email & password)

4. USER INPUT CREDENTIALS
   └─ User mengisi email dan password

5. BROWSER SUBMIT LOGIN
   └─ POST /login dengan data {email, password}

6. AUTH CONTROLLER CEK KREDENSIAL
   └─ Query ke database untuk validasi

7. DATABASE RETURN USER DATA
   └─ Jika valid: return {id, name, email, role}
   └─ Jika invalid: return NULL

8. CREATE SESSION
   └─ Jika valid, buat session untuk user

9. MIDDLEWARE CHECK ROLE
   └─ Memeriksa role user untuk redirect

10. ROLE-BASED REDIRECT
    ├─ Role 1 → /dashboard (Administrator)
    ├─ Role 2 → /dashboard-verifikator (Verifikator)
    ├─ Role 3 → /dashboard-operator (Operator Sekolah)
    └─ Role 4 → /kabalai-dashboard (Kepala Balai)

KREDENSIAL DEFAULT OPERATOR:

Email    : NPSN sekolah (contoh: 12345678)
Password : NPSN sekolah (contoh: 12345678)
Role     : 3 (Operator Sekolah)

Catatan: Operator disarankan mengganti password setelah login pertama kali

═══════════════════════════════════════════════════════════════════════════════════════════════════════════

SESSION DATA:

Setelah login berhasil, session menyimpan:
├─ user_id        : ID user
├─ name           : Nama user
├─ email          : Email user
├─ role           : Role user (1,2,3,4)
├─ sekolah_id     : ID sekolah (untuk operator)
└─ remember_token : Token untuk "Remember Me"

SECURITY:

✓ Password di-hash menggunakan bcrypt
✓ CSRF protection pada form login
✓ Session-based authentication
✓ Middleware untuk role-based access control
✓ Auto logout setelah inaktif (configurable)
